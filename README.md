## 简介

文件加密工具是一款专为个人和企业设计的文件安全软件，提供文件加密、解密、批量加密和文件粉碎四大核心功能。采用 RSA+AES 混合加密算法，确保您的敏感数据得到最高级别的保护。

---

## 功能介绍

### 🔒 单文件加密

选择任意文件进行加密处理，生成 `.encrypted` 加密文件。

**特点：**

- 支持 any 格式文件（文档、图片、视频、压缩包等）
- 加密后原文件保留，可自行决定是否删除
- 加密文件可安全传输、存储

**使用场景：**

- 加密重要合同、财务报表
- 保护个人隐私照片、视频
- 安全存储账号密码文档

---

### 📦 批量加密

将多个文件或整个文件夹打包加密为一个 `.encrypted` 文件。

**特点：**

- 支持同时选择多个文件
- 支持选择整个文件夹（保持目录结构）
- 一键加密，管理更便捷

**使用场景：**

- 加密整个项目文件夹
- 批量保护客户资料
- 打包加密多个敏感文档

---

### 🔓 文件解密

解密已加密的 `.encrypted` 文件，恢复原始文件。

**特点：**

- 自动识别单文件或批量加密格式
- 批量加密文件自动解压到独立文件夹
- 解密后文件完整保留原始内容

**使用方法：**

1. 选择要解密的 `.encrypted` 文件
2. 等待解密完成
3. 文件自动保存在同目录下

---

### 💀 文件粉碎

彻底删除文件，确保无法通过任何数据恢复工具找回。

**特点：**

- 3轮覆写：随机数据 → 零数据 → 随机数据
- 符合 DoD 5220.22-M 安全标准
- 覆盖后立即删除，无法恢复

**使用场景：**

- 销毁敏感商业文件
- 清理个人隐私数据
- 安全删除临时机密文档

> ⚠️ **警告**：文件粉碎为不可逆操作，请谨慎使用！

## 安全保障

### 加密技术

| 项目       | 技术参数                       |
| ---------- | ------------------------------ |
| 对称加密   | AES-256-CFB                    |
| 非对称加密 | RSA-2048                       |
| 密钥长度   | 256-bit (AES) / 2048-bit (RSA) |
| 加密模式   | 混合加密（RSA加密AES密钥）     |

### 加密流程

```
用户文件 → AES加密 → 加密数据
           ↓
    AES密钥 → RSA加密 → 安全存储
```

解密时需要对应的私钥才能还原文件，即使加密文件被他人获取，没有私钥也无法解密。

## 密钥管理

### 重要说明

程序首次运行时会在同目录下生成两个密钥文件：

| 文件              | 用途           | 重要性                   |
| ----------------- | -------------- | ------------------------ |
| `private_key.pem` | 私钥，用于解密 | ⚠️ **必须备份，切勿丢失** |
| `public_key.pem`  | 公钥，用于加密 | 可公开分享               |

### 安全建议

1. **备份私钥**：将 `private_key.pem` 复制到安全的存储位置（U盘、云盘等）
2. **保护私钥**：不要将私钥发送给他人或上传到公开平台
3. **丢失后果**：私钥丢失后将无法解密任何已加密的文件

### 多设备使用

如需在多台电脑上使用：

1. 将 `private_key.pem` 和 `public_key.pem` 复制到新电脑
2. 放置在与程序相同的目录下
3. 即可解密之前加密的文件

## 常见问题

### Q: 加密后的文件可以在其他电脑上解密吗？

**A:** 可以。只需将加密文件、程序和密钥文件（`private_key.pem`）一起复制到目标电脑即可解密。

### Q: 忘记备份私钥怎么办？

**A:** 非常抱歉，私钥丢失后将无法解密任何已加密的文件。这也是为什么我们强烈建议您在首次使用后立即备份私钥。

### Q: 文件粉碎后能恢复吗？

**A:** 不能。文件粉碎采用3轮覆写技术，符合安全标准，粉碎后的文件无法通过任何数据恢复工具找回。

### Q: 可以加密多大的文件？

**A:** 支持任意大小的文件，但对于超大文件（如几GB以上），加密时间会相应增加。

### Q: 加密会影响文件质量吗？

**A:** 不会。加密是无损的，解密后文件与原始文件完全一致，不会丢失任何数据。
